# IRBIS Client Feedback Sentiment Experiment

## Назначение проекта

Проект содержит эксперимент по обучению модели анализа тональности клиентских отзывов и обращений для веб-сервиса автоматизации взаимодействия с клиентами строительной компании. Эксперимент связан с темой выпускной квалификационной работы «Совершенствование процесса взаимодействия с клиентами компании с помощью веб-технологий».

Модель предназначена для обработки сообщений, поступающих через форму обратной связи, личный кабинет клиента или комментарии к заявкам. После анализа текста система определяет тональность обращения и выделяет тематический аспект: сроки, стоимость, документы, коммуникация, качество работ или объект строительства.

## Предметная область

Проект ориентирован на организацию ООО «ГК «ИРБИС»», работающую в сфере строительных проектов и технического надзора. В пользовательском приложении модель применяется как вспомогательный инструмент для менеджера: отрицательные и проблемные обращения можно быстрее находить в общем потоке заявок, а накопленные результаты использовать для аналитики качества клиентского взаимодействия.

## Используемый источник данных

В эксперименте использован открытый набор Geo Reviews Dataset 2023. Источник содержит отзывы об организациях, пользовательские оценки и рубрики. По рубрикам были отобраны записи, связанные со строительством, недвижимостью, жилыми комплексами, строительными магазинами, проектированием, ремонтом, инженерными услугами и смежными направлениями.

После предметной фильтрации, очистки и разметки был сформирован обучающий набор `irbis_feedback_dataset.csv`. Тональность определялась по оценке пользователя:

* оценки 1 и 2 — отрицательный отзыв;
* оценка 3 — нейтральный отзыв;
* оценки 4 и 5 — положительный отзыв.

Для уменьшения перекоса между классами выполнена балансировка. Итоговый обучающий набор содержит одинаковое количество записей для отрицательного, нейтрального и положительного классов.

## Этапы эксперимента

Эксперимент оформлен в ноутбуке `irbis_client_feedback_sentiment_experiment.ipynb` и включает восемь этапов.

1. Подготовка рабочей среды
   Создаются рабочие папки, фиксируется начальное число для случайных процедур, подключаются библиотеки.

2. Загрузка данных
   Данные загружаются онлайн в Google Colab из открытого источника. Полный исходный массив не сохраняется в проект как отдельный артефакт приложения.

3. Предметная фильтрация отзывов
   Из общего корпуса отбираются отзывы, связанные со строительной и смежной сферой.

4. Разметка и очистка отзывов
   Оценки преобразуются в классы тональности, тексты очищаются от технических символов и повторов.

5. Балансировка и сохранение обучающего набора
   Количество отзывов в классах выравнивается, после чего сохраняется файл `irbis_feedback_dataset.csv`.

6. Сравнение моделей разной сложности
   Проверяются наивный байесовский классификатор, логистическая регрессия, линейный метод опорных векторов, случайный лес и нейронная сеть LSTM.

7. Подбор параметров лучшей модели
   Для лучшей модели подбираются параметры преобразования текста и классификатора, затем выполняется итоговая проверка на контрольной части.

8. Сохранение артефактов для пользовательского приложения
   Сохраняются только файлы, нужные для интеграции в Flask-приложение.

## Результаты эксперимента

Лучший результат показал линейный метод опорных векторов с частотными признаками текста. После подбора параметров модель достигла следующих результатов на контрольной части:

| Показатель              | Значение |
| ----------------------- | -------: |
| Доля правильных ответов |   0.7165 |
| Средняя точность        |   0.7124 |
| Средняя полнота         |   0.7159 |
| Средняя F-мера          |   0.7132 |

Качество выше минимального порога 0.70, заданного для учебного интеллектуального сервиса. Положительные и отрицательные обращения распознаются устойчивее нейтральных. Нейтральный класс сложнее для модели, поскольку отзывы с оценкой 3 часто содержат смешанные формулировки: часть текста описывает недостатки, часть — приемлемые стороны услуги.

## Артефакты приложения

В архив пользовательского приложения включаются только необходимые файлы:

```text
models/model.joblib
models/model_info.json
models/aspect_keywords.json
requirements.txt
```

Файл `model.joblib` содержит итоговую модель. Файл `model_info.json` хранит классы, параметры, метрики и сведения о версии библиотек. Файл `aspect_keywords.json` содержит словарь аспектов клиентских обращений. Файл `requirements.txt` фиксирует зависимости для запуска веб-приложения.

Обучающий набор `data/processed/irbis_feedback_dataset.csv` нужен для проверки воспроизводимости эксперимента, но пользовательскому приложению он не требуется.

## Интеграция во Flask-приложение

Во Flask-приложении модель загружается при запуске сервера. Пользователь вводит текст отзыва или комментария к заявке, после чего приложение передает текст в сохраненную модель. Модель возвращает один из трех классов: `negative`, `neutral` или `positive`.

После определения тональности приложение применяет словарь аспектов и сохраняет результат в базу данных. Менеджер видит тональность обращения, аспект проблемы и может быстрее обработать сообщения, требующие внимания.

## Ограничения

Модель обучена на открытых отзывах об организациях, отобранных по строительным и смежным рубрикам. Такой источник приближает эксперимент к предметной области, но не заменяет реальные обращения клиентов ООО «ГК «ИРБИС»». После внедрения веб-сервиса качество можно повысить за счет накопления собственных заявок, комментариев и отзывов.

Нейтральные обращения требуют дополнительного внимания при интерпретации результата. Для управленческой аналитики их следует рассматривать как промежуточную категорию, а не как полностью беспроблемные сообщения.

## Запуск пользовательского приложения

Для запуска приложения необходимо перенести папку `models` и файл `requirements.txt` в корень Flask-проекта, установить зависимости и запустить сервер:

```bash
pip install -r requirements.txt
python main.py
```

После запуска приложение сможет загружать модель из файла `models/model.joblib` и применять ее к новым клиентским сообщениям.
