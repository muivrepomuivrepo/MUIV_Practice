# Веб-сервис автоматизации клиентских обращений ООО «ГК «ИРБИС»»

Проект представляет собой пользовательское Flask-приложение для автоматизации взаимодействия с клиентами строительной компании. Сервис позволяет принимать заявки, сопровождать обращения, хранить комментарии, собирать обратную связь и автоматически определять тональность клиентского сообщения с помощью заранее обученной модели.

Папка `models` в архиве содержит артефакты, полученные в Google Colab:

```text
models/model.joblib
models/model_info.json
models/aspect_keywords.json
```

Если файлы модели еще не добавлены, приложение запустится в демонстрационном режиме и покажет предупреждение в административной панели. После добавления артефактов перезапустите сервер.

## Основные возможности

- публичные страницы компании, услуг, проектов, контактов и справочной информации;
- регистрация и авторизация пользователей;
- три роли доступа: клиент, менеджер, администратор;
- личный кабинет клиента с заявками, комментариями и отзывами;
- панель менеджера для обработки заявок, смены статусов и анализа обратной связи;
- административная панель с контролем пользователей, услуг и состояния модели;
- SQLite-база данных для хранения пользователей, заявок, комментариев, отзывов и журнала действий;
- автоматическое определение тональности отзывов через `model.joblib`;
- определение тематического аспекта обращения через `aspect_keywords.json`;
- отчеты по статусам заявок, услугам, тональности и аспектам обращений.

## Установка

Рекомендуемая версия Python: 3.9.12 или выше в пределах той же ветки зависимостей, в которой была сохранена модель.

```bash
python -m venv .venv
```

Для Windows:

```bash
.venv\Scripts\activate
```

Для Linux и macOS:

```bash
source .venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

## Подключение модели

Добавьте в папку `models` три файла из архива, выгруженного из Google Colab:

```text
models/model.joblib
models/model_info.json
models/aspect_keywords.json
```

Файл `model.joblib` содержит обученную модель анализа тональности. Файл `model_info.json` хранит сведения о классах, метриках и параметрах. Файл `aspect_keywords.json` содержит словарь тематических аспектов.

## Запуск

```bash
python main.py
```

После запуска откройте адрес:

```text
http://127.0.0.1:5000
```

При первом запуске приложение создаст SQLite-базу данных и заполнит ее демонстрационными пользователями, услугами, заявками и отзывами.

## Работа интеллектуального модуля

При создании отзыва приложение выполняет три действия. Текст очищается от лишних пробелов и служебных символов, затем модель определяет тональность, после чего словарь аспектов выделяет тему обращения. Результат сохраняется в таблице отзывов и отображается менеджеру.

Если файл `model.joblib` отсутствует, применяется упрощенное правило на основе словарей. Такой режим нужен только для демонстрации интерфейса до подключения обученной модели.

## Назначение для ВКР

Проект можно использовать как практическую часть ВКР по теме «Совершенствование процесса взаимодействия с клиентами компании с помощью веб-технологий». Веб-сервис демонстрирует процесс обработки обращения: клиентская заявка, назначение услуги, комментарии, смена статуса, отзыв, анализ тональности и управленческая отчетность.
